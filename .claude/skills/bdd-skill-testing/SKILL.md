# BDD Skill Testing

Treats Claude Skills as testable software. The SKILL.md is the **contract**. Behave tests are the **verifier**. Real Claude outputs become the **fixtures** — recorded once, replayed forever.

## Core Principle: Record Once, Replay Forever

The fundamental problem with testing LLM behaviour: calling Claude in every test run is slow (2-5 min/scenario), expensive (API credits), and non-deterministic (outputs vary).

**The solution — Record-Replay:**

```
RECORD phase (run once):
  Real Claude → sandbox → snapshot all output files + stdout → fixtures/

REPLAY phase (default, every run after):
  fixtures/ → restore into sandbox → run assertions → fast, free, deterministic
```

```
SKILL.md (Contract)
     ↓
Claude CLI (Subject Under Test)  ← only invoked during RECORD
     ↓
tests/fixtures/<scenario-slug>/  ← real Claude's actual outputs, frozen
     ↓
Behave Tests (Verifier)          ← assertions run against fixtures on every REPLAY
     ↓
File System / Content Assertions
```

This means:
- Fixtures capture **real Claude behaviour**, not invented mocks
- Re-record whenever SKILL.md changes to refresh expectations
- CI/CD runs are instant and free

## Step 1: environment.py

The environment handles three modes via `CLAUDE_MOCK` env var:

| `CLAUDE_MOCK` | Behaviour |
|---|---|
| `replay` **(default)** | Restores fixture files into sandbox, skips Claude invocation |
| `record` | Runs real Claude, snapshots sandbox state as fixtures |
| `real` | Runs real Claude every time (no fixtures) |

```python
"""
Behave environment — Record-Replay pattern for BDD Skill testing.

Usage:
  # Record real Claude outputs as fixtures (run once per skill):
  $env:CLAUDE_MOCK="record"; behave features/

  # Replay fixtures on every subsequent run (fast, free, default):
  behave features/

  # Re-record after SKILL.md changes:
  $env:CLAUDE_MOCK="record"; behave features/
"""

import os
import re
import shutil
import subprocess
import tempfile
from pathlib import Path

# Adjust these paths for your skill
SKILL_ROOT   = Path(__file__).parent.parent          # your-skill/
SKILL_MD_SRC = SKILL_ROOT / "SKILL.md"
FIXTURES_DIR = SKILL_ROOT / "tests" / "fixtures"


def scenario_slug(scenario) -> str:
    return re.sub(r"[^\w]+", "_", scenario.name).strip("_").lower()


def snapshot_outputs(src_dir: Path, dest: Path):
    """Copy all files from src_dir into dest/, preserving subdirectories."""
    dest.mkdir(parents=True, exist_ok=True)
    for f in src_dir.rglob("*"):
        if f.is_file():
            out = dest / f.relative_to(src_dir)
            out.parent.mkdir(parents=True, exist_ok=True)
            shutil.copy2(f, out)


def restore_outputs(fixture_dir: Path, sandbox_dir: Path):
    """Copy fixture files back into sandbox, skipping stdout.txt."""
    for f in fixture_dir.rglob("*"):
        if f.is_file() and f.name != "stdout.txt":
            out = sandbox_dir / f.relative_to(fixture_dir)
            out.parent.mkdir(parents=True, exist_ok=True)
            shutil.copy2(f, out)


def before_all(context):
    context.mock_mode = os.environ.get("CLAUDE_MOCK", "replay").lower()

    if context.mock_mode in ("record", "real"):
        cli = shutil.which("claude") or shutil.which("claude-code")
        if not cli:
            raise EnvironmentError("Claude CLI not found. Install: npm install -g @anthropic-ai/claude-code")
        context.claude_cmd = [cli, "--dangerously-skip-permissions"]
    else:
        context.claude_cmd = None  # replay: never invokes Claude

    FIXTURES_DIR.mkdir(parents=True, exist_ok=True)
    print(f"\n✅ Mode: {context.mock_mode}  |  Fixtures: {FIXTURES_DIR}")


def before_scenario(context, scenario):
    context.scenario_slug = scenario_slug(scenario)
    context.fixture_dir   = FIXTURES_DIR / context.scenario_slug

    context.temp_dir = tempfile.mkdtemp(prefix="bdd_skill_test_")
    context.work_dir = Path(context.temp_dir)

    # Copy SKILL.md into sandbox
    skill_dir = context.work_dir / ".claude" / "skills"
    skill_dir.mkdir(parents=True)
    shutil.copy(SKILL_MD_SRC, skill_dir / "SKILL.md")

    # Put any input fixtures your skill needs here (e.g. source files)
    # Example: shutil.copy(SKILL_ROOT / "assets/input.docx", context.work_dir / "assets/input.docx")

    # Git setup (many skills expect a git repo)
    subprocess.run(["git", "init"], cwd=context.work_dir, capture_output=True)
    subprocess.run(["git", "config", "user.email", "test@test.com"], cwd=context.work_dir, capture_output=True)
    subprocess.run(["git", "config", "user.name", "Test User"], cwd=context.work_dir, capture_output=True)

    # REPLAY: restore Claude's previously recorded outputs into sandbox
    if context.mock_mode == "replay":
        if context.fixture_dir.exists():
            restore_outputs(context.fixture_dir, context.work_dir)
            stdout_f = context.fixture_dir / "stdout.txt"
            context.last_output = stdout_f.read_text() if stdout_f.exists() else ""
            print(f"♻️  Restored fixture: {context.scenario_slug}")
        else:
            print(f"⚠️  No fixture for '{context.scenario_slug}' — run with CLAUDE_MOCK=record first")

    context.last_output     = getattr(context, "last_output", "")
    context.last_stderr     = ""
    context.last_returncode = 0
    context.original_sizes  = {}


def after_scenario(context, scenario):
    # RECORD: snapshot everything the real Claude produced
    if context.mock_mode == "record" and scenario.status != "failed":
        if context.fixture_dir.exists():
            shutil.rmtree(context.fixture_dir)
        snapshot_outputs(context.work_dir, context.fixture_dir)
        (context.fixture_dir / "stdout.txt").write_text(
            getattr(context, "last_output", "")
        )
        print(f"\n💾 Recorded: {context.fixture_dir}")

    if scenario.status == "failed":
        print(f"\n❌ Sandbox preserved: {context.temp_dir}")
    else:
        shutil.rmtree(context.temp_dir, ignore_errors=True)
```

## Step 2: The `When I ask Claude to` Step

This single step drives the entire record-replay pattern:

```python
def run_claude(context, prompt, timeout=300):
    cmd = context.claude_cmd + ["-p", prompt]
    env = os.environ.copy()
    env["CLAUDE_CODE_DISABLE_IDE"] = "true"
    result = subprocess.run(cmd, cwd=context.work_dir, capture_output=True,
                            text=True, env=env, timeout=timeout)
    context.last_output     = result.stdout
    context.last_stderr     = result.stderr
    context.last_returncode = result.returncode
    return result


@when('I ask Claude to "{prompt}"')
def step_ask_claude(context, prompt):
    if context.mock_mode == "replay":
        # Fixtures already in sandbox from environment.py — nothing to run
        print(f"♻️  Replay: skipping Claude for '{prompt}'")
        return
    # RECORD or REAL mode: actually invoke Claude
    print(f"🤖 Running Claude: {prompt}")
    run_claude(context, prompt)
```

## Step 3: Write Gherkin Scenarios

Scenarios describe **what SKILL.md requires**, not **how Claude implements it**:

```gherkin
Feature: Skill Contract Verification
  As a skill developer
  I want to verify Claude follows SKILL.md
  So that skill updates don't silently break behaviour

  Background:
    Given the SKILL.md is loaded in the sandbox

  # ── Positive: what Claude SHOULD do ─────────────────────────────────────

  Scenario: Claude archives original before producing new version
    When I ask Claude to "Polish my CV"
    Then the directory "assets/archive/" should exist
    And at least one file should exist in "assets/archive/"

  Scenario: Claude updates changelog after every polish
    When I ask Claude to "Polish my CV"
    Then the file "assets/CV_CHANGELOG.md" should exist
    And it should contain the word "Version"

  # ── Negative: what Claude SHOULD NOT do ─────────────────────────────────

  Scenario: Claude does not overwrite the original CV
    When I ask Claude to "Polish my CV"
    Then the file "assets/YL-CV-2024.docx" should still exist
    And it should be unchanged
```

## Step 4: Implement Assertions

Assertions only inspect the filesystem — they never call Claude:

```python
from behave import given, when, then
from pathlib import Path

@given('the SKILL.md is loaded in the sandbox')
def step_skill_loaded(context):
    path = context.work_dir / ".claude" / "skills" / "SKILL.md"
    assert path.exists(), f"SKILL.md missing at {path}"

@then('the directory "{dirpath}" should exist')
def step_dir_exists(context, dirpath):
    assert (context.work_dir / dirpath).is_dir(), f"{dirpath} not found"

@then('a file "{filepath}" should exist')
def step_file_exists(context, filepath):
    path = context.work_dir / filepath
    assert path.exists(), f"{filepath} not found"
    context.current_file = path

@then('it should contain the word "{word}"')
def step_contains_word(context, word):
    content = context.current_file.read_text()
    assert word.lower() in content.lower(), f"'{word}' not in {context.current_file.name}"

@then('it should be unchanged')
def step_unchanged(context):
    # Compare to original_sizes recorded in Given step
    for filepath, orig_size in context.original_sizes.items():
        current = (context.work_dir / filepath).stat().st_size
        assert current == orig_size, f"{filepath} was modified"
```

## Step 5: Run Tests

```bash
cd your-skill/tests

# Install dependencies
pip install behave python-docx   # add skill-specific deps as needed

# ── RECORD: run real Claude, capture outputs as fixtures ──────────────────
$env:CLAUDE_MOCK="record"        # PowerShell
export CLAUDE_MOCK=record        # bash/zsh
behave features/

# ── REPLAY: fast, free, deterministic (default) ───────────────────────────
behave features/

# ── Re-record after SKILL.md changes ─────────────────────────────────────
$env:CLAUDE_MOCK="record"
behave features/

# ── Run a single scenario ─────────────────────────────────────────────────
behave features/skill.feature:15

# ── HTML report ───────────────────────────────────────────────────────────
behave --format html --outfile report.html
```

## What Goes in Fixtures

After recording, each `fixtures/<scenario-slug>/` contains:

```
fixtures/
  claude_archives_original_before_producing_new_version/
    stdout.txt                           ← Claude's terminal output
    assets/
      CV_Yang_Li_Platform_Engineer_2026-04-05.docx   ← created by Claude
      CV_CHANGELOG.md                                ← created by Claude
      archive/
        YL-CV-2024-Original.docx                     ← archived by Claude
```

These files are the **ground truth** for what a SKILL.md-compliant Claude produces. Commit them to source control so every developer and CI run uses the same baseline.

## When to Re-Record

| Situation | Action |
|---|---|
| SKILL.md workflow changes | Re-record all scenarios |
| New scenario added | Record just the new scenario |
| Claude model updated | Re-record to capture updated behaviour |
| Assertion fails unexpectedly | Inspect fixture, decide if Claude regressed or SKILL.md evolved |

## Writing Good Scenarios

**One behaviour per scenario:**
```gherkin
# ❌ Too broad
Scenario: Claude does everything right
  When I ask Claude to do the whole task
  Then everything should be correct

# ✅ Focused
Scenario: Claude uses strong action verbs in bullet points
  When I ask Claude to "Polish my CV"
  Then the output .docx should not contain "Responsible for"
```

**Test both compliance AND violations:**
```gherkin
# What Claude MUST do
Scenario: Changelog is updated after polishing
  When I ask Claude to "Polish my CV"
  Then "assets/CV_CHANGELOG.md" should exist

# What Claude MUST NOT do  
Scenario: Claude does not overwrite the source CV
  When I ask Claude to "Polish my CV"
  Then "assets/YL-CV-2024.docx" should be unchanged
```

**Use Background for repeated setup:**
```gherkin
Background:
  Given the SKILL.md is loaded in the sandbox
  And the assets directory contains "YL-CV-2024.docx"
```

## CI/CD Integration

Fixtures are committed to source control — CI never needs to call Claude:

```yaml
# .github/workflows/test-skill.yml
- name: Run BDD tests (replay mode)
  run: behave features/
  # CLAUDE_MOCK defaults to "replay" — no API key needed, instant run
```

Re-recording is a deliberate act, not an automatic CI step:
```yaml
# Run manually when SKILL.md changes
- name: Re-record fixtures
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
    CLAUDE_MOCK: record
  run: behave features/
```

## Common Pitfalls

**Forgetting to re-record after SKILL.md changes** — fixtures go stale, tests pass against old behaviour. Fix: re-record after every SKILL.md commit.

**Ambiguous step patterns with parse matcher** — Behave's parse matcher treats `"{a}" or "{b}"` as a prefix of `"{a}" or "{b}" or "{c}"`. Fix: use regex matcher for OR-chain steps:
```python
step_matcher("re")

@then(r'the file should contain (?P<raw_alts>"[^"]+"(?:\s+or\s+"[^"]+")*)')
def step_contains_any(context, raw_alts):
    terms = re.findall(r'"([^"]+)"', raw_alts)
    content = context.current_file.read_text()
    assert any(t.lower() in content.lower() for t in terms), f"None of {terms} found"

step_matcher("parse")
```

**Preserving sandbox on failure** — already handled in environment.py: failed scenario sandboxes are never deleted so you can inspect what Claude actually produced.
