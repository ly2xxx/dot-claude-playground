## How to Use This File

Each time you create a new CV version or make significant changes:

1. **Add a new entry** with:
   - Version number
   - Date
   - Target (General/Job-specific/Industry-specific)
   - Summary of changes

2. **Example entry format:**

```markdown
### v2.1 - 2026-03-25 - Tailored for Google Senior Engineer

**Changes:**
- Emphasized scale (10M+ users, 99.9% uptime)
- Added Kubernetes and GCP keywords
- Reordered projects to highlight distributed systems work
- Quantified all performance improvements with metrics
- Adjusted summary to mention "large-scale systems"

**File:** `CV_Yang_Li_Google_Senior_Engineer_2026-03-25.docx`
```

## Benefits of Version Tracking

✅ **Easy rollback** - Find what worked in previous successful applications  
✅ **Pattern recognition** - See which changes correlate with interview invites  
✅ **Avoid repetition** - Remember what you already tried  
✅ **Collaboration** - If working with career coach, they can see change history

## Tips

- Update this file BEFORE applying to job (forces you to review changes)
- Note which versions got you interviews (helps refine strategy)
- Keep this file even after landing job (helps with future job searches)
