# Tech/Software Engineering CV Guide

**Industry-specific best practices for technology, software engineering, and technical roles.**

---

## Tech Industry CV Priorities

1. **Technical impact & scale** (served X users, processed Y requests)
2. **Technologies used** (specific languages, frameworks, tools)
3. **Open source / side projects** (demonstrates passion beyond 9-5)
4. **Problem-solving** (complexity of challenges overcome)
5. **Collaboration** (cross-functional, remote-friendly)

---

## What Tech Recruiters Look For

### Must-Have Section: Technical Skills

**Format:**
```
TECHNICAL SKILLS

Languages: Python, JavaScript, TypeScript, Go, SQL
Frameworks/Libraries: React, Node.js, FastAPI, Django, TensorFlow
Cloud/DevOps: AWS (EC2, S3, Lambda), Docker, Kubernetes, Terraform, GitLab CI/CD
Databases: PostgreSQL, MongoDB, Redis, Cassandra
Tools: Git, Jira, Confluence, VS Code, Postman
```

**Rules:**
- List 8-15 skills (not 50 - shows lack of depth)
- Order by proficiency (strongest first) OR category
- Only include skills you can discuss in interview
- Match job description keywords

---

## Achievement Writing for Tech Roles

### Emphasis: Scale + Impact + Technology

**Template:**
```
[Action verb] [technology/system] [complexity indicator] resulting in [scale/impact metric]
```

**Examples:**

1. **Backend Engineering:**
   > Architected and deployed microservices-based API handling 10M+ daily requests with 99.95% uptime, reducing latency from 500ms to 80ms using Golang, Redis caching, and AWS Lambda

2. **Frontend Engineering:**
   > Built responsive React dashboard serving 50K daily users, reducing page load time by 65% through code splitting, lazy loading, and CDN integration across 30+ components

3. **DevOps/SRE:**
   > Designed and implemented CI/CD pipeline using GitLab and Docker, reducing deployment time from 2 hours to 8 minutes and enabling 10x more frequent releases with zero downtime

4. **Data Engineering:**
   > Built real-time data pipeline processing 5TB daily using Apache Spark and Kafka, reducing data latency from 24 hours to 15 minutes and enabling same-day business insights

5. **Machine Learning:**
   > Developed recommendation engine using TensorFlow and deployed on AWS SageMaker, serving 2M users and increasing product engagement by 35% (A/B tested, p<0.01)

**Key elements:**
- ✅ Specific technology stack
- ✅ Scale metrics (users, requests, data volume)
- ✅ Performance improvement (latency, speed, efficiency)
- ✅ Business impact (revenue, users, engagement)

## GitHub & Portfolio Strategy

### Include GitHub Profile

**In contact section:**
```
Email@example.com | linkedin.com/in/yourname | github.com/yourusername
```

**When to include:**
- If you have 5+ meaningful repositories
- If contributing to open source projects
- If showing side projects

**When to skip:**
- If your GitHub is empty or only forks
- If your projects are outdated (4+ years old)
- If your code quality is poor

---

### Portfolio Website (Optional)

**Include if:**
- You're in frontend/design-adjacent role
- You have impressive visual projects
- You want to show personality/branding

**URL format:** yourname.dev or yourname.com (professional, not yourname.blogspot.com)

## FAANG-Specific Optimization

### What FAANG Looks For

1. **Scale:** Did you work with millions of users/requests?
2. **Impact:** Did you move key business metrics?
3. **Complexity:** Did you solve hard technical problems?
4. **Leadership:** Did you influence/mentor others?
5. **Ownership:** Did you drive projects end-to-end?

### FAANG CV Formula

**For each bullet:**
```
[Ownership action verb] [complex system/project] [scale indicator] resulting in [business impact]
```

**Example:**
> Spearheaded migration of monolithic application to microservices architecture (15 services, 5M lines of code) serving 50M users, reducing deployment risk and enabling 3 new product launches in 6 months (previously impossible)

**Google's XYZ Formula:**
> Accomplished [X] as measured by [Y], by doing [Z]

**Example:**
> Improved search relevance by 12% (measured by user click-through rate) by implementing deep learning ranking model trained on 100M query-result pairs using TensorFlow

## Startups vs Big Tech: CV Adjustments

### For Startups

**Emphasize:**
- Ownership ("Built from scratch", "Led initiative")
- Scrappiness ("With limited resources", "Self-directed")
- Versatility ("Wore multiple hats", "Full-stack")
- Speed ("Launched in 4 weeks", "Rapid iteration")
- Equity/stake (if applicable): "Joined as employee #5"

**Example:**
> As first engineering hire, single-handedly built MVP product (React, Django, PostgreSQL) serving initial 5K users, securing $2M seed funding and enabling team growth to 10 engineers

---

### For Big Tech

**Emphasize:**
- Scale ("10M users", "1B requests/day")
- Collaboration ("Cross-functional team of 20")
- Best practices ("Designed for 99.99% uptime")
- Systems thinking ("Scalable architecture")
- Mentorship ("Mentored 3 junior engineers")

**Example:**
> Architected and led implementation of distributed caching layer using Redis, serving 100M daily requests across 5 global data centers, reducing API latency by 60% and improving user experience for flagship product

## Technical Interview Preparation (CV Related)

### Be Ready to Discuss

For EVERY technology/project on your CV:
- What problem were you solving?
- Why did you choose that technology?
- What were the trade-offs?
- What would you do differently now?
- What was the hardest bug/challenge?

**Red flag:** Listing technologies you can't discuss deeply.

## Common Mistakes in Tech CVs

### ❌ Mistake 1: Listing Every Technology You've Touched

**Bad:**
```
Skills: Python, Java, JavaScript, C++, C#, Ruby, PHP, Go, Rust, Kotlin, Swift, R, MATLAB, HTML, CSS, SQL, NoSQL, GraphQL, REST, SOAP...
```

**Why bad:** Dilutes expertise. If you list 30 languages, recruiter assumes you're "beginner" in most.

**Good:**
```
Languages: Python (expert), JavaScript/TypeScript (proficient), Go (proficient), SQL
```

---

### ❌ Mistake 2: No Quantification

**Bad:**
> Developed features for mobile app using React Native

**Good:**
> Built 15 user-facing features for React Native mobile app serving 500K monthly users, including push notifications (20% engagement increase) and offline mode

---

### ❌ Mistake 3: Acronym Overload Without Context

**Bad:**
> Utilized K8s, GCP GKE, and TF for IaC deployment via GL CI/CD pipeline for MSA handling API reqs

**Good:**
> Deployed microservices architecture on Google Kubernetes Engine (GKE) using Terraform for infrastructure-as-code, automated via GitLab CI/CD pipeline handling 1M+ daily API requests

---

### ❌ Mistake 4: Old/Irrelevant Technologies

**Remove (unless job specifically requires):**
- Flash, Silverlight, Internet Explorer 6 support
- PHP 4, Python 2 (unless maintaining legacy)
- Technologies not used in 5+ years
- "Microsoft Office" (assumed for tech roles)

---

### ❌ Mistake 5: "Responsible for" Instead of Action

**Bad:**
> Responsible for maintaining CI/CD pipeline

**Good:**
> Optimized CI/CD pipeline reducing build time from 45 minutes to 8 minutes, enabling 5x more frequent deployments

## Tech CV Length Guidelines

| Experience Level | CV Length |
|------------------|-----------|
| 0-3 years | 1 page |
| 3-8 years | 1-2 pages (prefer 1) |
| 8-15 years | 2 pages |
| 15+ years | 2 pages (focus on last 10-12 years) |

**Exception:** Academia/research with publications = CV (3-5 pages acceptable)

## Education Section for Tech

### Recent Grads (<2 years)

**Include:**
- GPA (if >3.5)
- Relevant coursework
- Technical projects
- Hackathons/awards

**Example:**
```
Bachelor of Science in Computer Science | University Name | 2023
GPA: 3.8/4.0
Relevant Coursework: Machine Learning, Distributed Systems, Algorithms, Database Systems
Awards: First Place - University Hackathon 2023 (team of 4, built real-time collaboration tool)
```

---

### Experienced (5+ years)

**Keep minimal:**
```
Master of Science in Computer Science | University Name | 2018
Bachelor of Science in Computer Science | University Name | 2015
```

**Skip:** GPA, coursework, projects (professional experience is more relevant)

---

### Self-Taught / Bootcamp Grads

**Emphasize projects and experience over education:**

```
EDUCATION

Full-Stack Web Development Bootcamp | Coding Academy | 2022
• 12-week intensive program: JavaScript, React, Node.js, PostgreSQL
• Capstone Project: Built e-commerce platform handling 5K products

Bachelor of Arts in Economics | University Name | 2018
```

## Remote Work Indicators

**If applying to remote roles, signal remote readiness:**

- "Remote" in job title: "Senior Engineer | Remote"
- Mention remote collaboration tools: "Collaborated with distributed team across 5 time zones using Slack, Zoom, and Jira"
- Highlight async communication skills
- Show self-direction: "Self-directed", "Autonomous"

## Tech CV Keywords (ATS Optimization)

### Common Keywords by Role

**Backend Engineer:**
- API, REST, GraphQL, microservices, scalability, database optimization, caching, message queues, CI/CD

**Frontend Engineer:**
- React, Vue, Angular, responsive design, performance optimization, accessibility, cross-browser, webpack, state management

**DevOps/SRE:**
- CI/CD, Kubernetes, Docker, AWS, infrastructure-as-code, Terraform, monitoring, observability, automation

**Data Engineer:**
- ETL, data pipeline, Spark, Kafka, Airflow, data warehouse, data lake, SQL, NoSQL, big data

**Machine Learning:**
- TensorFlow, PyTorch, scikit-learn, model training, feature engineering, A/B testing, MLOps, deployment

## Final Checklist: Tech CV

- [ ] Technical Skills section lists 8-15 relevant technologies
- [ ] Every achievement includes specific technology used
- [ ] At least 70% of bullets are quantified (users, scale, performance)
- [ ] GitHub profile included (if quality repos)
- [ ] No outdated technologies (unless job requires)
- [ ] All projects can be discussed in depth
- [ ] Action verbs are technical ("Architected", "Deployed", "Optimized")
- [ ] Scale indicators included ("10M users", "1B requests")
- [ ] CV length appropriate for experience level
- [ ] ATS-friendly formatting (no fancy graphics/templates)

---

**Remember:** Tech recruiters scan for 3 things in 30 seconds:
1. Can you code in required languages?
2. Have you worked at scale?
3. Can you solve complex problems?

Make those three things impossible to miss.
